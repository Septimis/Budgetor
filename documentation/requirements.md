# Requirements - Budgetor
### Introduction
Budgetor will be a web application designed to deliver budgeting and financial tracking needs to end-users.  The web application will eventually be converted over to an application run by iOS and Android.

### 1. Roles & Goals
- **1.1** User
    - **1.1.1:** Easily be able to add, remove, and modify budget categories.
    - **1.1.2:** Easily be able to view relavent information regarding the end-users finances in a convenient and understandable manner such as graphs or numbers.
    - **1.1.3:** Be alerted when an untracked transaction is found and prompted to categorize it.
### 2. Functional Requirements
- **2.1** Intuitive design that allows a user to create, remove, and modify a budget category.
    - **2.1.1:** Allow a user to create, remove, and modify budget category blocks that may house different budget categories.  (i.e. A 'Transportation' block that contains the budget categories, 'Gas', 'Car Insurance', 'Bus Fares', and 'Repairs/Maintenance')
- **2.2** Importing transactions from a user's financial institution.
    - **2.2.1:** Import the meta data from a transaction such as the time the transaction was completed, to/from whom the money was transacted from, and the amount exchanged.
- **2.3** Be able to assign a budget category associated with a certain transaction and add an optional comment.
    - **2.3.1:** A transaction must be assigned a budget category.
### 3. Non-Functional Requirements
- **3.1** Toolset:
    - **3.1.1:** html
    - **3.1.2:** .css
    - **3.1.3:** javascript
    - **3.1.4:** dotnet core
    - **3.1.5:** GitHub
    - **3.1.6:** Git
### 4. Future Features:
- **4.1** Create an application where the budget can be viewed & modified on an iOS or Android device.
- **4.2** Implement additional features such as credit score calculator, etc.
